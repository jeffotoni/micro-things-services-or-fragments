# Micro things, serviços ou fragmentos.

Uma pergunta feita no grupo [Microservices Brasil](https://t.me/microservicesbr) do telegram se tornou umas das discussões mais frutiferas que já presenciamos. Estando como ouvinte, eu, @lucasbadico, me prontifiquei a documentar e levantar as liçoes dessa discução, e jogar de novo na comunidade para podermos evoluirmos o conceito.

## Principios

- Não somos donos da verdade, não queremos ditar como deve ser ou não feito alguma coisa.
- Acreditamos que questionar, colocar a prova, conceitos já aceitos como verdades e talvez desenterrar alguns vistos como mentiras podem enriquecer toda a comunidade.
- Esse projeto não é nosso modo de cravar uma resposta, mas de levantar perguntas para respondermos todos juntos.
- Respeito, inclusive para com aqueles que não nos respeitam, se farpas vierem a serem enviadas, não as retornem.

## Proposta de como desenvolveremos as discussões

Levantarei um issue para cada tópico, uma vez q tenhamos um desenho dos pontos de vista, não precisamos concordar, mas termos claro as abordagens, alguem abre um merge request sobre aquele ponto. Na forma de um capitulo em um livro.

Algo importante também é compartilharmos nossas arquiteturas, o @jeffotoni mesmo diz que não gosta de ficar só na teoria, que gosta de prática, então além do capitulo, seria bom termos no merge request codigo de alguma, ou algumas implementações das soluções propostas.
